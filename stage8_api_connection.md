# 🔗 Этап 8 — Подключение Lovable к API
## Пошаговый план

> **Цель:** Все разделы ЛК вызывают реальный API на `api.drivegpt.pro`
> **Сейчас:** Lovable работает с мок-данными (фейковые данные внутри)
> **После:** Lovable берёт данные с сервера и отправляет на сервер

---

## 📋 Порядок подключения (8 шагов)

| # | Раздел | API | Сложность |
|---|--------|-----|:---------:|
| 1 | Настройка базового URL | — | 🟢 Лёгкая |
| 2 | Авторизация (вход) | `POST /api/auth/login` | 🟡 Средняя |
| 3 | Проекты | `GET/POST/DELETE /api/projects` | 🟢 Лёгкая |
| 4 | Настройки (Telegram Token) | `PUT /api/projects/{id}/token` | 🟢 Лёгкая |
| 5 | Python-код (загрузка/управление) | `POST upload`, `POST start/stop` | 🟡 Средняя |
| 6 | Диалоги | `GET dialogs`, `POST send` | 🟡 Средняя |
| 7 | База данных (пользователи) | `GET users`, `POST import` | 🟡 Средняя |
| 8 | Логи заявок | `GET requests` | 🟢 Лёгкая |

---

## Шаг 1. Настроить базовый URL API

**Промпт для Lovable:**
```
Создай файл конфигурации API. Все запросы к бэкенду должны идти через 
единый базовый URL.

Создай константу API_BASE_URL = "https://api.drivegpt.pro"

Создай вспомогательную функцию apiRequest(method, path, body) которая:
- Добавляет базовый URL к пути
- Добавляет заголовок Authorization: Bearer {token} из localStorage
- Устанавливает Content-Type: application/json
- При ответе 401 — перенаправляет на страницу входа
- При ошибке — показывает toast с текстом ошибки
- Возвращает JSON-ответ

Пример использования:
  const projects = await apiRequest("GET", "/api/projects")
  await apiRequest("POST", "/api/projects", { name: "Мой бот" })
```

**Что произойдёт:** Появится единая точка для всех API-запросов.

## Шаг 2. Подключить авторизацию

> ⚠️ **Важно:** Перед этим шагом нужно добавить JWT-авторизацию на сервер. 
> Пока её нет — можно пропустить этот шаг и подключать разделы без авторизации.

**Промпт для Lovable:**
```
Подключи страницу входа к API.

При нажатии кнопки «Войти»:
1. Отправить POST /api/auth/login с телом { email, password }
2. При успехе — сохранить token в localStorage
3. Перейти на главную страницу (список проектов)
4. При ошибке — показать красный текст под формой

При выходе:
1. Удалить token из localStorage
2. Перенаправить на страницу входа

Базовый URL API: https://api.drivegpt.pro
```

## Шаг 3. Подключить раздел «Проекты»

**Промпт для Lovable:**
```
Подключи раздел «Проекты» к реальному API.

Загрузка списка проектов:
- При открытии страницы: GET /api/projects
- Отобразить карточки из ответа (id, name, status, bot_username, created_at)
- Если status = "active" — зелёный бейдж "running"
- Если status = "inactive" или "stopped" — серый бейдж "stopped"

Создание проекта:
- Кнопка «+ Новый проект» → модалка с полем «Название»
- При подтверждении: POST /api/projects?name={название}
- После создания — обновить список проектов

Удаление проекта:
- Кнопка «Удалить» на карточке → подтверждение «Вы уверены?»
- При подтверждении: DELETE /api/projects/{id}
- После удаления — обновить список

Открытие проекта:
- Кнопка «Открыть» — сохранить project_id и показать подменю 
  (Python-код, Диалоги, База данных, Логи заявок, Настройки)

Базовый URL: https://api.drivegpt.pro
Убрать все мок-данные (фейковые проекты).
```

## Шаг 4. Подключить «Настройки» (Telegram Token)

**Промпт для Lovable:**
```
Подключи раздел «Настройки» к API.

Telegram Bot Token:
- При открытии: GET /api/projects/{project_id} — показать текущий токен 
  (замаскированный: ***XXXX)
- При нажатии «Сохранить токен»: 
  PUT /api/projects/{project_id}/token?token={введённый_токен}
- При успехе — показать информацию о боте:
  «Бот: @username (Имя бота)» + зелёная галочка
- При ошибке — показать красный текст «Неверный токен»

Управление ботом:
- Кнопка «Запустить»: POST /api/projects/{project_id}/start
- Кнопка «Остановить»: POST /api/projects/{project_id}/stop 
- Статус: GET /api/projects/{project_id}/status
  Ответ содержит: status, webhook_url, error_message

Webhook URL:
- Показать из ответа /status: webhook_url
- Кнопка «Копировать» рядом

Базовый URL: https://api.drivegpt.pro
```

## Шаг 5. Подключить «Python-код»

**Промпт для Lovable:**
```
Подключи раздел «Python-код» к API.

Загрузка кода:
- При drag-and-drop ZIP: POST /api/projects/{project_id}/upload
  (отправить как FormData, поле file)
- Показать спиннер во время загрузки
- При успехе — обновить дерево файлов

Дерево файлов:
- При открытии раздела: GET /api/projects/{project_id}/files
- Показать список файлов из ответа (path, size)
- При клике на файл: GET /api/projects/{project_id}/files/{path}
  Показать содержимое в модалке (моноширинный шрифт)

Проверка синтаксиса:
- Кнопка «Проверить»: POST /api/projects/{project_id}/check
- Если ok=true → зелёный текст «✅ Синтаксис корректен»
- Если ok=false → красный текст с ошибками из errors[]

Логи бота:
- GET /api/projects/{project_id}/logs?lines=50
- Показать в блоке логов (моноширинный шрифт)
- Кнопка «Обновить логи»

Базовый URL: https://api.drivegpt.pro
```

## Шаг 6. Подключить «Диалоги»

**Промпт для Lovable:**
```
Подключи раздел «Диалоги» к API.

Список диалогов (левая панель):
- При открытии: GET /api/projects/{project_id}/dialogs
- Каждый элемент: user_id, username, first_name, last_message, 
  last_message_at, message_count
- Сортировка по last_message_at (новые сверху)

История чата (центральная панель):
- При клике на диалог: GET /api/projects/{project_id}/dialogs/{user_id}
- Показать messages[] как пузыри чата:
  direction="in" → слева (от пользователя)
  direction="out" → справа (от бота)
- sent_by показать label: "user" или "bot" или "operator"

Отправка от имени бота:
- При отправке: POST /api/projects/{project_id}/dialogs/{user_id}/send?text={текст}
- После отправки — обновить историю чата

Профиль пользователя (правая панель):
- При клике на имя: GET /api/projects/{project_id}/users/{user_id}
- Показать карточку с полями из ответа

Убрать все мок-данные диалогов.
Базовый URL: https://api.drivegpt.pro
```

## Шаг 7. Подключить «Базу данных»

**Промпт для Lovable:**
```
Подключи раздел «База данных» к API.

Таблица пользователей:
- При открытии: GET /api/projects/{project_id}/users
- Заполнить таблицу из ответа: id, telegram_id, username, first_name, 
  email, generations_count, generations_balance, first_purchase, created_at
- Поле «Положительный баланс» — показать ✅ если generations_balance > 0

Карточка пользователя:
- При клике на строку: GET /api/projects/{project_id}/users/{user_id}
- Показать панель / модалку с полными данными

Экспорт CSV:
- Кнопка «Экспорт»: GET /api/projects/{project_id}/users/export
- Скачать полученный файл

Импорт CSV:
- После загрузки файла и маппинга полей:
  POST /api/projects/{project_id}/users/import
  (отправить как FormData с файлом + JSON маппинга)
- Показать результат: сколько записей добавлено

Убрать все мок-данные пользователей.
Базовый URL: https://api.drivegpt.pro
```

## Шаг 8. Подключить «Логи заявок»

**Промпт для Lovable:**
```
Подключи раздел «Логи заявок» к API.

Таблица заявок:
- При открытии: GET /api/projects/{project_id}/requests?page=1&per_page=25
- Ответ содержит: items[], total, page, per_page, pages

Фильтр по датам:
- При изменении дат: 
  GET /api/projects/{project_id}/requests?date_from=2026-03-01&date_to=2026-03-10&page=1&per_page=25
- При выборе периода (Сегодня/Вчера/Неделя/Месяц) — вычислить даты 
  и подставить в запрос

Пагинация:
- Использовать total и pages из ответа
- «Показано {(page-1)*per_page+1}-{min(page*per_page, total)} из {total}»
- При клике на номер страницы — запросить с новым page=N

Фото:
- car_photo_url и wheel_photo_url — показать как кликабельные ссылки
- При клике — открыть фото в новой вкладке

Убрать все мок-данные заявок.
Базовый URL: https://api.drivegpt.pro
```
