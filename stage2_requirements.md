# Этап 2. Расширение функциональности

## 2.1 Управление пользовательскими переменными и тегами
Переменная «Количество генераций»должна изменяться в зависимости от Заявок. Если в логе появилась Заявка от пользователя, то переменная должна поменяться на то значение, которое указано в Заявке (ниже. п.2.1.1 Логи заявок)
Теги: «Положительный баланс» и «Первая покупка» должны быть активными полями (галочка включена / выключена). Теги должны меняться через код бота и вручную через карточку пользователя в Базе данных.
Переменная с генерацией и теги должны отображаться в карточке пользователя в Базе данных и их можно корректировать вручную, убавляя и прибавляя значения, а также выставляя нужные теги пользователю.

Добавить кнопку Сохранить (по стилю как кнопка Скачать шаблон), которая появляется после любых изменений данных. После нажатия - кнопка меняет цвет на Зеленый и текст: Успешно ✅, а после пропадает.

### 2.1.1 Новое поле в разделе Логи заявок
Добавить поле/столбик Стоимость между Датой и ID

С помощью кода, мы будем устанавливать стоимость каждой заявки (не всегда 1 заявка = 1 генерация) и записывать это значение в поле Стоимость.
Далее поле исходя их значение в поле Стоимость должно меняться количество оставшихся генераций у пользователя.

## 2.4 Интеграция платежной системы (обсудить сложность)
Добавить интеграцию платежной системы ЮKassa. Кнопки тарифов размещаются в коде бота. При нажатии на тариф формируется ссылка на оплату. 

## 2.5 Обработка оплаты
После успешной оплаты начисляются генерации в зависимости от пакета. Обновляется тег «Положительный баланс». Каждому новому пользователю присваивается признак “новый пользователь”. Если совершается покупка — этот признак убирается. 
Сохраняется email пользователя. (Это скорее информация для составления корректной документации API)

## 2.6 Интеграция мессенджера MAX
Добавить поддержку мессенджера MAX. Доки: https://dev.max.ru/docs-api 
Интеграция реализуется по аналогии с Telegram.

## 2.7 Раздел «Статистика» (РАЗДЕЛ пока для ознакомления, чуть позже будем реализовывать)
Добавить раздел статистики для каждого проекта.
Показатели: общее количество генераций, общее количество пользователей, количество новых пользователей.
Фильтрация и выборка на периоды: сегодня, вчера, неделя, месяц, квартал, год, произвольный период.
Группировка данных: по минутам, по 10 минутам, по часам, по дням, по неделям, по месяцам.
Добавить флажок «Уникальные значения». Если флажок активен — учитывается только первая заявка пользователя.

## 2.8 Раздел «База данных» (скорее инфа для API документации)
Добавить в документацию правила общения кода с Базой данных. Чтобы когда мы загрузили в базу пользователей, с помощью Импорта с нашего текущего сервиса Лидтех - мы могли к ней обращаться через код. 
Чтобы проверять при запуске бота есть ли текущий пользователь в базе и если да - то запускать соответствующий сценарий.
