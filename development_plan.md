| Было | Стало | Почему |
|------|-------|--------|
| Новый отдельный VPS | **Существующий сервер** (yookassa-api) | Экономия |
| 8 ГБ RAM | **2 ГБ RAM** | Достаточно для 1 пользователя и 2 ботов |
| Docker-контейнеры | **Virtualenv + systemd** | Экономия RAM (~2× меньше) |
| PostgreSQL | **SQLite** | Экономия RAM, проще настройка |
| Отдельный сервер для DriveGPT | **На одном сервере с YooKassa** | Экономия 550 ₽/мес |

## 🧩 Как устроен сервис — простыми словами

```
┌─────────────────────────────────────────────────┐
│            Сервер 185.178.47.148                 │
│                                                  │
│  ┌────────────────┐  ┌───────────────────┐       │
│  │  YooKassa API  │  │  DriveGPT Backend │       │
│  │  (уже работает)│  │    (FastAPI)      │       │
│  └────────────────┘  └────────┬──────────┘       │
│                               │                  │
│                    ┌──────────┴──────────┐       │
│                    │  Webhook-роутер      │       │
│                    │  Принимает события   │       │
│                    │  из Telegram и       │       │
│                    │  передаёт боту       │       │
│                    └──────────┬──────────┘       │
│                               │                  │
│           ┌───────────────────┼──────────┐       │
│           │                   │          │       │
│   ┌───────┴──────┐   ┌───────┴──────┐   │       │
│   │   Бот 1      │   │   Бот 2      │   │       │
│   │ (virtualenv) │   │ (virtualenv) │   │       │
│   └──────────────┘   └──────────────┘   │       │
│                                          │       │
│   ┌──────────────────────────────────┐   │       │
│   │  SQLite (база данных)            │   │       │
│   │  Пользователи, диалоги, заявки   │   │       │
│   └──────────────────────────────────┘   │       │
│                                                  │
│  Nginx → drivegpt.pro (HTTPS)                    │
└──────────────────────────────────────────────────┘
```

**Как работает:**
1. Пользователь пишет боту в Telegram
2. Telegram отправляет сообщение на `drivegpt.pro/webhook/id-проекта`
3. Nginx принимает → передаёт FastAPI-серверу
4. FastAPI определяет, какому боту предназначено → передаёт дальше
5. Python-бот обрабатывает и отвечает пользователю
6. Все данные (диалоги, заявки, пользователи) сохраняются в SQLite
7. В личном кабинете видно: кто написал, что ответил бот, какие заявки

#### Шаг 2.2 — Настроить Nginx
На сервере создать файл конфигурации:

Я предоставлю содержимое этого файла — нужно будет скопировать его в терминал.

Затем:
```bash

systemctl reload nginx
```


### День 4 (5 марта) — Webhook-роутер + менеджер ботов

> **Цель:** Самое важное — принимать сообщения из Telegram и управлять ботами.

#### Шаг 4.1 — Webhook-роутер + авто-регистрация пользователей

```
Telegram → POST drivegpt.pro/webhook/{project_id}
                    │
                    ▼
            FastAPI принимает
                    │
                    ▼
         Проверяет пользователя в БД
         (по Telegram ID из update)
                    │
              ┌─────┴──────┐
              │            │
         Есть в БД    Нет в БД
              │            │
              │      Создаёт карточку:
              │      • Имя (из TG)
              │      • TG ID
              │      • Username
              │      • Внутр. ID (авто)
              │      • Генерации = 0
              │      • 1-я покупка = Нет
              │            │
              └─────┬──────┘
                    │
                    ▼
         Передаёт Python-боту
         (через HTTP на внутренний порт)
                    │
                    ▼
         Бот обрабатывает + отвечает
```

> **Важно:** Сервис **автоматически** создаёт карточку пользователя при первом сообщении. Код бота НЕ должен заботиться о регистрации — это делает webhook-роутер.

#### Шаг 4.2 — Менеджер ботов (без Docker)

Вместо Docker используем **virtualenv + systemd**:

**Запуск бота:**
1. Код бота лежит в `/opt/drivegpt/bot-projects/project-123/`
2. Создаётся virtualenv в этой папке
3. Устанавливаются зависимости из `requirements.txt`
4. Проверяется синтаксис Python (`python -m py_compile`)
5. Создаётся systemd-сервис для автозапуска
6. Регистрируется webhook в Telegram API

**Остановка бота:**
1. Останавливается systemd-сервис
2. Удаляется webhook из Telegram

#### Шаг 4.3 — Загрузка кода

| Действие | URL | Что делает |
|----------|-----|-----------|
| Загрузить | POST `/api/projects/{id}/upload` | ZIP с кодом |
| Дерево файлов | GET `/api/projects/{id}/files` | Список файлов |
| Содержимое файла | GET `/api/projects/{id}/files/{path}` | Текст файла |
| Проверка | POST `/api/projects/{id}/check` | Проверка синтаксиса |

- Доступны по ссылке `https://drivegpt.pro/files/abc123.jpg`
- Автоудаление через 7 дней (cron-задача раз в сутки)

#### Шаг 5.4 — Внутренний API для кода бота

API, которое код бота использует для записи данных в сервис:

| Действие | URL | Зачем |
|----------|-----|-------|
| Обновить данные | PUT `/internal/users/{tg_id}` | Изменить баланс, email и т.д. |
